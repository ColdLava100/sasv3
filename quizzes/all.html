🏆 Topic 1: Rank vs Filter — Basic

Q1: Show products where Sales > $10,000. Filter or Rank?

✅ Filter — Condition: Sales > $10,000. Rows not matching are removed.
❌ Rank: Rank picks Top/Bottom N by a measure, not by a condition threshold.

Q2: "Show Top 10 best-selling products." Filter or Rank?

✅ Rank — "Top N by a measure" = Rank.
❌ Filter: No condition like "Sales > X" is specified. You want the top-scoring items, not items matching a condition.

Q3: Can a Rank's count be controlled by a Parameter + Slider?

✅ Yes! Connect a slider to a numeric parameter, use that parameter as the rank count. User slides → rank updates dynamically.
❌ No: This is actually a common SAS VA feature — parameters can drive rank counts interactively!

🏆 Topic 1: Rank vs Filter — Advanced

Q4 🔥: "Show Top 10 products by Profit, but only for the Sports category." What do you need?

✅ Both! First Filter to keep only Sports, then Rank Top 10 by Profit from what's left.
❌ Filter only: Filter keeps Sports but doesn't rank — you'd get all sports products, not just Top 10.
❌ Rank only: Rank doesn't filter — you'd get Top 10 across ALL categories, not just Sports.

Q5 🔥: A manager wants "Bottom 5 stores by Profit Margin." You also need to exclude stores with less than $1,000 total sales first. What's the order of operations?

✅ Filter first, then Rank — Always filter before ranking! Remove stores with sales < $1,000 (Filter), then find Bottom 5 from the remaining stores (Rank).
❌ Rank first: You'd rank ALL stores including the ones with low sales, potentially getting wrong results.
❌ Either order: The order matters — Filter removes data, Rank works on what remains.

Q6 🧠: Which statement is TRUE about Rank vs Filter?

✅ Filter uses a condition — "Does each row match my condition? Yes→keep, No→remove."
❌ Rank removes rows: Yes, Rank DOES remove rows (only shows Top/Bottom N). Filter also removes rows.
❌ Filter picks Top 5: That's what Rank does. Filter uses conditions like "Sales > $10,000."
❌ Rank reorders: Sort reorders. Rank reduces to Top/Bottom N — it also removes rows.

📐 Topic 3: Columns, Rows & Lattice — Basic

Q1: A crosstab has Region in Columns, Year in Rows. Where does "Asia" appear?

✅ Across top as column header — Region is in the Columns role, so values like "Asia" spread across the top.
❌ Down side: That's the Rows role. Year would be there instead.
❌ Inside a cell: That's where Measures (values) appear, not categories.

Q2: Which objects use Lattice Columns/Rows?

✅ Charts only — Lattice Columns/Rows are for chart panels. Crosstabs use their own Columns/Rows data roles, which are different!
❌ Crosstabs: They use regular Columns/Rows roles, not Lattice.
❌ Both: No object uses both sets — they're mutually exclusive by object type.

Q3: A bar chart has three panels side by side (North, South, East). What role is "Region"?

✅ Lattice Columns — Side-by-side panels = Lattice Columns.
❌ Lattice Rows: That's for stacked (vertical) panels.
❌ Group: Group controls color, not panel arrangement.

📐 Topic 3: Columns, Rows & Lattice — Advanced

Q4 🔥: Scatter plot with two panels: Heavy (top), Moderate (bottom). Smoking Status role?

✅ Lattice rows — Stacked vertically = Lattice Rows.
❌ Lattice columns: Would be side-by-side, not stacked.
❌ Color: Color would assign different colors per value, not create separate panels.
❌ Filter: Filter would remove data, not split into panels.

Q5 🔥: Why can't a Crosstab use Lattice Columns?

✅ Lattice is for charts — They're fundamentally different! Crosstab Columns/Rows = data categories that create grid headers. Lattice Columns/Rows = chart panels that split visual space.
❌ "It can": No, the roles are designed for different object types and can't be swapped.
❌ "Planned fix": Not a bug — it's by design.

Q6 🧠: If you had a crosstab and wanted to split it into side-by-side mini-tables by Region, what would you use?

✅ Lattice Columns (but crosstabs can't use Lattice) — This reveals the key difference! Crosstabs can't create panel layouts. You'd need to use separate report pages or a page prompt to simulate the effect.
❌ Both Columns and Rows: Would create a grid, not side-by-side panels.
❌ Page prompt: Can filter but doesn't create side-by-side views.

📋 Topic 5: Crosstab Rows Role

Q1: Which data item types can go in the Rows role of a crosstab? (Choose all that apply)

✅ Category + Single hierarchy — From course: "Both Columns and Rows accept either a single hierarchy or any number of categories."
❌ Measure: Measures go in the Measures role (cells), not in Rows or Columns.
❌ Multiple hierarchies: Only ONE hierarchy allowed in Rows at a time.

Q2: What's the difference between a Category and a Single Hierarchy in Rows?

✅ Category = no drill-down; Single Hierarchy = drill-down — A hierarchy (like Year→Quarter→Month) enables drill-down. A plain category (like Product) just shows one level.
❌ Same thing: Categories and hierarchies are different data item types with different capabilities.
❌ Memory: Not about memory — about structure and drill-down behavior.

Q3 🔥: Can Measures ever go in the Rows role of a Crosstab?

✅ No — Measures NEVER go in Rows or Columns roles. They belong in the Measures role, which fills the data cells of the crosstab.
❌ Aggregated: Even aggregated measures stay in the Measures role, not Rows.

📋 Topic 6: Measure Heading Placement — Basic

Q1: Changing Measure heading placement from ABOVE to LEFT does what?

✅ Moves Frequency label to the rows side — Only the measure name label moves from above the columns to the left side.
❌ Transposes: Transposing would swap Make↔Origin entirely. Only the LABEL moves.
❌ Origin moves: Origin stays as columns — only the measure label moves.

Q2: What does Measure heading placement NOT change?

✅ Data item role assignments — It ONLY moves the measure label. Make still in Rows, Origin still in Columns.
❌ Where the label appears: That IS what it changes — it moves the label from above to the left.

📋 Topic 6: Measure Heading Placement — Advanced

Q3 🔥: True or False: Changing Measure heading placement redesigns the entire crosstab by swapping which categories are in rows vs columns.

✅ False — It ONLY moves the measure label. The category role assignments (which data items are in Rows vs Columns) stay EXACTLY the same.
❌ True: This is the #1 misconception! Only the word "Frequency" changes position — nothing else.

Q4 🧠: You have Make in Rows, Origin in Columns, Frequency as measure. Someone says "just transpose it." What's the minimum change needed to switch the view?

✅ Swap data items — To truly transpose you must reassign the data items: Move Make from Rows to Columns, Origin from Columns to Rows. Measure heading placement only moves the label — it's NOT transposition!
❌ Heading placement: That only moves the label, doesn't swap categories.
❌ Aggregation type: Doesn't change layout at all.

📦 Topic 7: Box Plot — Basic

Q1: What does the horizontal LINE (──) inside the box represent?

✅ Median - LiMe LINE = MEDIAN. The median is the middle value.
❌ Mean: The mean is shown as a SQUARE, only if Averages option is ON.
❌ Std dev: Not shown as a line in the box.
❌ Q1: Q1 is the bottom edge of the box.

Q2: What does the black SQUARE inside the box represent?

✅ Mean - SqMean SQUARE = MEAN. Only with Averages option ON.
❌ Median: The median is the LINE inside the box.
❌ Mode: Not shown in standard box plots.
❌ Q3: Q3 is the top edge of the box.

Q3: What SAS option turns ON the black square mean marker?

✅ Averages - Adds mean marker for comparing mean vs median.
❌ Outliers: Controls how extreme values display Show/Hide/Ignore.
❌ Display Rules: Conditional formatting thresholds.
❌ Box Direction: Horizontal vs vertical orientation.

Q4: If LINE median and SQUARE mean are FAR APART, what does that mean?

✅ Data is skewed - Outliers pull the mean away from the median.
❌ Normal: If normal, mean median, so they would be close together.
❌ Box too small: Box size IQR is independent of mean-median distance.

Q5: What are whiskers in a box plot?

✅ Acceptable range lines - Like a cats whiskers extending from its face!
❌ Decorative: Whiskers have statistical meaning - data range within 1.5 IQR.
❌ Connecting lines: Each box plot is independent.

📦 Topic 7: Box Plot — Advanced

Q6: Which outlier setting is the DEFAULT in SAS box plots?

✅ Hide Outliers - By default outliers are hidden. Included within whiskers.
❌ Show: Must manually toggle to see outlier dots.
❌ Ignore: Must manually toggle to exclude outliers.

Q7: With Show Outliers what happens to the whiskers?

✅ Stop at 1.5 IQR - Dots or gray bins shown for outliers beyond.
❌ Full range: That is Hide Outliers whiskers extend to include outliers.
❌ Disappear: Whiskers are always present unless IQR = 0.

Q8: Three box plots same data. BP1 has gray outlier bins. BP2 looks normal. BP3 has no box. Which is Hide Outliers?

✅ BP2 normal - Gray bins = SHOW. Normal = HIDE default. No box = IGNORE.
❌ BP1: Gray area means outlier bins visible = SHOW not Hide.
❌ BP3: No box means IQR = 0 because outliers excluded = IGNORE.
❌ None: Since Hide is DEFAULT the normal box IS hiding outliers.

Q9: Outliers are included within the whiskers means what?

✅ Whiskers extend to cover outlier values - Outliers absorbed into whisker range.
❌ Shrink: That would hide outliers by cutting them off not including.
❌ Part of box: Outliers dont affect IQR only the whisker range.

Q10: What does MSRP stand for?

✅ Manufacturers Suggested Retail Price - Car sticker price from SAS data.
❌ Market: Manufacturer not Market.
❌ Maximum: Wrong terms entirely.

🟦 Topic 9: Treemap — Basic

Q1: In a treemap what defines the actual tiles/groups?

✅ Category - Each unique category value becomes a tile.
❌ Size: Controls tile area not which tiles exist.
❌ Color: Controls color gradient not tile definition.
❌ Frequency: Default for Size when no measure assigned.

Q2: What determines tile SIZE in a treemap?

✅ Size measure or Frequency - A measure controls tile area. If none assigned defaults to Frequency count.
❌ Category: Category defines WHICH tiles not their size.
❌ Color: Color is independent of size.

Q3: What type of chart is a treemap?

✅ Part-to-whole - Like pie/donut charts shows how parts contribute to a whole.
❌ Relationship: Scatter plots show relationships.
❌ Time series: Line charts show trends over time.
❌ Distribution: Histograms show distribution.

Q4: Profit is assigned to Color. If Profit is removed from Color what happens?

✅ All tiles same color - No Color assignment = no data for colors. Color and Size are independent roles.

🗺️ Topic 10: Geo Maps — Basic

Q1: A Geo Bubble map uses which geometry type?

✅ Points ● — Geo Bubble, Geo Coordinate, and Geo Pie are all point maps.
❌ Polylines ═══: Used for Geo Line, Geo Network, Geo Line-Coordinate (paths/routes).
❌ Polygons █: Used for Geo Region, Geo Cluster, Geo Contour (filled regions).

Q2: A Geo Line map uses which geometry type?

✅ Polylines ═══ — Geo Line and Geo Network display paths/routes.
❌ Points ●: Used for Geo Bubble, Geo Coordinate, Geo Pie (pinpointing locations).
❌ Polygons █: Used for Geo Region maps (filling regions with color).

Q3: A Geo Region map uses which geometry type?

✅ Polygons █ — Geo Region fills regions with color (choropleth map).
❌ Points ●: Can't show regions — only dots at specific coordinates.
❌ Polylines ═══: Can't fill areas — only draw paths between points.

Q4: What are the 3 ways to create a geography data item?

✅ Geographic name/code lookup, Latitude & Longitude in data, Geographic data provider
❌ Measure/Category/Parameter: These are data item types, not methods for creating geography data items.

Q5: Which type of geo map is NOT a point map?

✅ Geo Line — It's a polyline map (paths/routes).
❌ Geo Bubble: Point map — bubbles at coordinates, sized by a measure.
❌ Geo Coordinate: Point map — dots at coordinates.
❌ Geo Pie: Point map — mini pie charts at coordinates.

Q6: Which map type combines polygons + points in one view?

✅ Geo Region-Coordinate — Fills regions AND shows coordinate dots.
❌ Geo Region: Polygons only — no coordinate dots.
❌ Geo Cluster: Groups regions — doesn't show individual points.
❌ Geo Bubble: Points only — no polygon regions.

🗺️ Topic 10: Geo Maps — Advanced

Q7 🔥: Sales data across US states with uneven distribution — California has $50M but Delaware has $2M. Which point map is BEST?

✅ Geo Bubble — From course: "Geo bubble maps are useful when data is not evenly distributed." Bubble size represents the measure, so Delaware doesn't look as big as California.
❌ Geo Coordinate: Should only be used with evenly distributed data. Uniform dots would make tiny and big states look equal.
❌ Geo Pie: Used for showing distribution among a few groups per location, not for comparing a single measure across many regions.

Q8 🔥: Display flight routes between airports. The route path matters AND you want airport names on hover. Which type?

✅ Geo Line-Coordinate — Shows route paths (polylines) AND coordinate markers at endpoints with hover info.
❌ Geo Line: Shows paths but no interactive dots or hover labels on endpoints.
❌ Geo Network: For interconnected nodes with many-to-many connections, not simple routes.
❌ Geo Bubble: Bubbles show measures at points — can't show route paths.

Q9 🔥: Data has store addresses as Lat/Lon. You want regions colored by total sales. What can you use?

✅ Both B and C — You CANNOT use Geo Region with Lat/Lon!
❌ Geo Region: Course says: "Geography data items created using latitude and longitude in data cannot be used in geo region maps."
✅ Geo Coordinate (dots colored by sales) and Geo Bubble (bubbles sized by sales) both work with Lat/Lon data items.

Q10 🔥: A Geo Network shows connections between nodes. What must be TRUE about source & target data items?

✅ Both must be geographic data items — Course: "If both the source and target nodes are geographic data items, then the network can be overlaid on a geo map."
❌ Both measures: Networks need geographic locations, not numeric values.
❌ Different data sources: They can be from the same source — what matters is geographic classification.

Q11 🔥: What permissions are needed to create a geographic data provider? Why does this matter for geo line maps?

✅ Special user permissions required — Course: "Special user permissions are required to define and edit geographic data providers." Geo line maps need polylines, which often come from a geographic data provider (custom routes). Without permission, you can't create custom polyline data!

Q12 🔥: Show profit by country (colored regions) + store locations (dots) in one map. Which type?

✅ Geo Region-Coordinate — Combines polygon regions (colored by profit) WITH coordinate dots (store locations) in a single map.
❌ Geo Region: Shows regions but NO coordinate dots.
❌ Geo Bubble: Shows points but NO filled regions.
❌ Two separate maps: Possible but unnecessary when Geo Region-Coordinate exists!

Q13 🔥: Sales territory groupings — custom clusters not matching state lines. Which combination?

✅ Geo Cluster + Geographic data provider — Course: "To use custom polygonal shapes, a geographic data provider must be defined." Standard lookups won't work for custom territories.
❌ Geo Region + Lookup: Lookup only knows standard boundaries (countries, states), not custom clusters.
❌ Geo Bubble + Lat/Lon: Shows points, not region groupings.

Q14 🔥: A Geo Contour map is BEST for what?

✅ Density patterns — Contour lines show areas of equal value/density (like elevation or heat maps).
❌ Exact sales: Use Geo Region (colored states/countries) for exact values per region.
❌ Store locations: Use Geo Coordinate or Geo Bubble for individual points.
❌ Airline routes: Use Geo Line or Geo Line-Coordinate for paths between locations.

Q15 🔥: Identify the FALSE statement about geo maps:

✅ FALSE: Lat/Lon works with ALL types — This is wrong! Lat/Lon CANNOT be used with Geo Region maps.
✅ Geo Bubble + uneven data: TRUE — from course: "Geo bubble maps are useful when data is not evenly distributed."
✅ Geo Coordinate + even data: TRUE — from course: "Geo coordinate maps should be used only when there is an even distribution of values."
✅ Provider from ESRI/CAS: TRUE — from course: "The geographic data provider can be created from either an Esri feature service or a CAS table."

Q16 🏆: Supply chain dashboard: distribution centers as bubbles, delivery routes, sales regions colored. Minimum geo maps needed?

✅ 1 — custom geo map layers all three — Course: "You can mix and match these elements to create your custom geo map." A custom geo map in Graph Builder can layer Geo Map (polygons) + Scatter Plot (points) + Bubble Plot (bubbles) + Line Plot (routes) all in one.

🔀 Path Analysis — Link Color Options

Q1: What is a Path Analysis diagram also called?

✅ Sankey diagram — Course: "This type of diagram is also called a Sankey diagram."
❌ Flow map: Geo maps show flow, not path analysis.
❌ Network graph: Network analysis is different — shows relationships, not sequential flows.
❌ Tree map: Shows part-to-whole relationships, not flows.

Q2: Which Link color option gives each unique journey its own distinct color from start to end?

✅ Path — "Displays each path as a separate group with a distinct color."
❌ Drop off: Only 2 colors — continuing vs dropping.
❌ Event: Color by the node/stage type, not by path.
❌ Sequence: Color by position number, not by path identity.

Q3 🔥: A Sankey has each path with its OWN color throughout. Which option is set?

✅ Path — The exam question! Each path having its own color = Path coloring.
❌ Drop off: Would show only 2 colors — "continuing" vs "drop-off."
❌ Event: All paths at the same stage would share the same color.
❌ Sequence: All 1st steps would share one color, all 2nd steps another.

Q4: What determines link WIDTH in a path analysis?

✅ Weight measure or frequency — "Width of link represents frequency or weight measure."
❌ Link color: Color controls grouping, not width.
❌ Number of nodes: Node count doesn't affect link thickness.

Q5: Which data role in Path Analysis uniquely labels each object (e.g., customer ID)?

✅ Transaction identifier — "Uniquely labels each object to analyze."
❌ Event: Defines the nodes/stages.
❌ Sequence order: Orders the events.
❌ Weight: Link width.

Q6 🧠: "Each Path gets its own _______."

✅ Paint job (color) — "Each Path gets its own Paint job" 🎨
❌ Drop off: Only determines if a link terminates or not.
❌ Event: Defines the node/stage.
❌ Width: Determined by weight measure or frequency.

🎯 Parameters & Expressions

Q1: What is a Parameter in SAS VA?

✅ A variable whose value can be changed — Course: "A parameter is a variable whose value can be changed by report viewers."
❌ Formula: That's an Expression (calculated item).
❌ Chart: Parameters are data items, not visual objects.
❌ Data source: Parameters are created within a report.

Q2: Which control object can NOT populate a Numeric Parameter?

✅ List control — "List control can be used with a multiple-value character parameter." Numeric parameters use: Slider, Text input, Drop-down, Button bar.
❌ Slider: IS available for numeric params.
❌ Text input: IS available.
❌ Button bar: IS available.

Q3: What type of parameter would you use for a "Select Region" dropdown?

✅ Character — Regions are text values. "If you want viewer to choose a category or enter text, create a character parameter."
❌ Numeric: Region names aren't numbers.
❌ Date: Regions aren't dates.
❌ Expression-based: This uses a formula, not user selection.

Q4 🔥: You want an "Average Profit" value that is NOT affected by report filters. What should you use?

✅ Expression-Based Parameter — "Expression-based parameters are NOT affected by filters or actions — based on entire data source, value remains constant."
❌ Calculated Item: Calculated items operate on filtered data, so the average would change when filtered.
❌ Numeric Parameter: User sets this manually, not from data.
❌ Character: Stores text, not calculated averages.

Q5: Where can a Character parameter be used?

✅ Anywhere character values make sense — "Character parameters can be used in calculated items, aggregated measures, filters, URLs, and text objects."
❌ Calc items only: Much broader — filters and URLs too.
❌ Ranks only: Ranks typically use numeric parameters.
❌ Display rules only: Display rules accept numeric params mostly.

Q6 🧠: "Parameters are the _____. Expressions are the _____."

✅ Input / Calculation — "Parameters are the INPUT. Expressions are the CALCULATION. They work TOGETHER."
❌ Calculation/Input: Opposite! Parameters receive input, expressions calculate.
❌ Chart/Data: Neither is a chart.
❌ Filter/Rank: Both can USE parameters, but that's not what they are.

Q7 🔥: What happens when you use a Numeric Parameter in a Rank's "Count" field?

✅ The rank shows TOP N where N = parameter value — "Numeric parameters can be used in ranks for the n or count value." If parameter = 10, Rank = Top 10. Slide to 25, Rank = Top 25.
❌ Stores rank result: Parameters are input, not output.
❌ Can't be used: They absolutely can — it's a key use case!

📤 Sharing & Embedding Reports

Q1: What is Guest Access in SAS VA?

✅ Anonymous Read-Only access — "Guest Access provides anonymous Read-Only access to reports."
❌ Paid: Guest Access is an optional admin-configured feature, not a paid tier.
❌ Mobile editing: Guest is read-only — no editing.

Q2: What does an exported Report Package contain?

✅ A snapshot of data and query results — "You can export a report package, which is a snapshot of the data and query results for a report."
❌ Live link: A package is static/offline — no live Viya connection.
❌ Chart images only: It includes data too.

Q3 🔥: Two methods to embed a report on a website with NO viewer sign-in?

✅ Exported Report Package + Direct Connection with Guest Access — These are the EXAM QUESTION correct answers!
❌ PDF + Copy Link: PDF is static (good) but Copy Link alone requires sign-in unless Guest Access is enabled.

Q4 🔥: Why is "Exported Report Package WITH Guest Access enabled" WRONG?

✅ Guest Access is for LIVE connections — Report Package is static — The exam trap! Guest Access is a feature of SAS Viya live connections. An exported package is offline (no Viya connection), so "Guest Access" doesn't apply to it. The package alone already needs no sign-in.
❌ Admin approval: True but that's not why it's wrong.
❌ Can't be hosted: They CAN be hosted on webservers.

Q5: Who must enable Guest Access?

✅ An administrator — "An administrator must create a new configuration in SAS Environment Manager for the sas.guest service."
❌ Viewer: Viewers can't enable it — it's an admin-level config.
❌ Designer: Regular designers don't have admin permissions.

Q6: What prevents PDF and Report Package exports?

✅ Private data — "If a report contains private data, then exporting to PDF and exporting the report package are disabled."
❌ Too many pages: Not a limitation.
❌ No charts: Text-based reports can still be exported.

Q7: Which method generates code for the SAS Viya SDK for JavaScript?

✅ Copy Embeddable Markup — "Generates a code snippet for use in the SAS Viya SDK for JavaScript to embed content in a custom web page."
❌ Copy Link: Gives a URL link, not code/snippet.
❌ Report Package: Exports a static file, not embed code.

Q8 🧠: "Package = _____. Guest = _____."

✅ Passive / Granted — "Package = Passive (no connection needed). Guest = Granted (admin must enable)."
❌ Active/Denied: Opposite meanings.
❌ Online/Offline: Package is offline, Guest is for online.

🌳 Decision Tree

Q1: Which statement describes how a decision tree is used?

✅ Segments data into branches to predict — The EXAM question! "Creates decision rules to predict the response variable."
❌ Histograms: Descriptive stats.
❌ Form: Tree learns from data.
❌ Scatter: Correlation analysis.

Q2: What two roles are MANDATORY for a decision tree?

✅ Response + Predictors — Response = what to predict. Predictors = explaining variables.

Q3: Why are decision trees "white box" models?

✅ Transparent decision-making — "Structure is easily visualized, decision-making is transparent and understandable."

Q4: Which license for advanced decision tree features?

✅ SAS Visual Statistics — Advanced features: predictor importance, assessment plots, interactive mode.

Q5 🔥: What can you do in Interactive Mode on a decision tree?

✅ Prune, edit splits, train — Right-click to enter. Cut nodes (prune), change split rule, or grow tree.

Q6: What does the Icicle Plot show?

✅ Alternative view of the tree — Selecting a node in one view highlights it in the other.

🧮 Calculated Data Items

🧮 Calculated Data Items

Q1: What is the key difference between a Calculated Data Item and an Aggregated Measure?

✅ Calculated = detail (row-by-row); Aggregated = aggregated (after grouping)
❌ Opposite: That's backwards.

Q2: Which requires an aggregation function (SUM, AVG, COUNT)?

✅ Aggregated Measure — Must have an aggregation function. Calculated items do NOT require aggregation.

Q3: What are the two contexts for an Aggregated Measure expression?

✅ By Group and For All — By Group = per subset. For All = entire data (after filtering).

Q4 🔥: An aggregated measure "% diff from previous period" was created using Year. In a crosstab with City in Rows, it shows dashes. What's needed in Columns?

✅ Year — The period dimension must be in the table structure for the periodic function to work.
❌ Quarter/Month/Date: Wrong period type or too granular.

Q5 🔥: Which periodic function calculates "same month, last year"?

✅ ParallelPeriod — "Returns the aggregated value for a period parallel to the current period."

Q6: Can an Aggregated Measure be used inside another Aggregated Measure?

✅ No — "Aggregated functions do NOT accept aggregated measures as inputs (no nested aggregations)."

🔄 Common Filters

🔄 Common Filters

Q1: What types of filters can become a Common Filter?

✅ Basic, Advanced, or Post-Aggregate — "A common filter can be a basic, advanced, or post-aggregate filter."

Q2: Where does a Common Filter appear after creation?

✅ Data pane under Common Filter group — Appears in the Data pane and can be applied via Filters pane.

Q3 🔥: Common filters are tied to a data source and shared between objects. What are two MORE attributes? (Choose 2)

✅ A) Available to entire report AND D) Edits apply everywhere
❌ B) Only on page: FALSE — available to ENTIRE report (across pages).
❌ C) Admins: FALSE — any user can create common filters.

Q4: How do you change a common filter for only ONE object?

✅ Revert to object-specific copy — "You can revert the common filter to an object-specific copy" for one-off changes.

Q5: A Common Filter is available to what scope?

✅ The entire report (all pages) — "Available to the entire report (yes, across pages)." The #1 exam trap!

🤖 Auto Explanation & Prediction

🤖 Automated Explanation & Prediction

Q1 🔥: In the Automated Explanation object, what happens when you CLICK on an underlying factor in the bar chart?

✅ Relationship plot is displayed — "Click on an underlying factor to view the relationship plot."
❌ Removed: Right-click removes, left-click shows relationship.
❌ New prediction: Clicking doesn't create objects.

Q2: What happens when you RIGHT-CLICK an underlying factor?

✅ The factor is removed — "You can remove any individual underlying factor from the right-click context menu."
❌ Relationship plot: That's left-click.

Q3: What models does Automated Prediction run for a CATEGORY response?

✅ Logistic regression, Gradient boosting, Decision tree — Category response models.
❌ Linear regression: Used for MEASURE responses.

Q4: How does Automated Prediction choose the champion model?

✅ Category: highest accuracy; Measure: lowest ASE — "Champion Model based on highest accuracy for category, lowest ASE for measure."
❌ Manual: "No manual model selection."

Q5: Which data items are NOT automatically added as underlying factors?

✅ Aggregated measures and date/time values — Also excluded: calculated items, custom categories, derived items.

Q6: Can Automated Prediction be a source for actions?

✅ No — "Cannot use as source or target of actions" is a key limitation.

📦 Content Objects

📦 Content Objects

Q1: Which Content Object can be the source of a filter action?

✅ Data-Driven Content — "Can even be the source of an action." Web and Job Content cannot be sources of filter actions.
❌ Web/Job: "Cannot be the source of a filter action: images, job content, text, and web content."

Q2: Which Content Objects are NOT included in PDF export?

✅ Web Content AND Data-Driven Content — "Data-driven content objects and web content objects are not included in the PDF."
❌ Web only: Data-Driven also excluded.

Q3: What does the Web Content object use to display external pages?

✅ An iFrame — "Display web page, embed video in an iFrame." Some websites block iFrame embedding (x-frame-options=sameorigin).

Q4 🔥: Which Content Object receives data from Visual Analytics in JSON format?

✅ Data-Driven Content — "Data messages (data assigned to object and metadata information) are sent from VA to the data-driven content object in JSON format."

Q5: Which data items does the Web Content object require?

✅ None — just a URL — Web Content displays an external web page. It doesn't use any data items from VA.

Q6: What is required to use the Data-Driven Content object?

✅ HTML, CSS, and JavaScript — "To use the data-driven content object, you will need some knowledge of HTML, CSS, and JavaScript."
❌ SAS job: That's for Job Content.
❌ URL: That's for Web Content.

Q7 🧠: "Web = ____. Job = ____. Data-Driven = ____."

✅ URL / SAS Code / HTML+Data — Web = URL link, Job = pre-built SAS code, Data-Driven = custom code + live VA data.
