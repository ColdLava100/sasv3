🙈 Hidden Concepts

Q1: What does the Hidden Data Role do?

✅ Includes in query, not displayed — "The hidden data role enables you to include a data item in the query results when you want to use its values but do not want the data item displayed."
❌ Hides page: That's Hidden Page, a different concept.

Q2: A Hidden Page is visible to whom?

✅ Designers, not viewers — "Hidden pages are visible to report designers as a page tab, but not to report viewers." Viewers CAN access it via a link though.

Q3: Can a Hidden Page be accessed by report viewers?

✅ Yes, via a link — "In view mode, a linked hidden page will appear like a regular page." It's hidden from the tab bar, but accessible if navigated to.

Q4 🔥: What is NOT included in PDF export?

✅ Hidden Pages — "Hidden pages are not included in the PDF." Hidden Data Role items and hidden columns ARE included if their objects are exported.
❌ All: Only Hidden Pages are excluded from PDF.

Q5: What is Hide Column in a list table?

✅ Shortcut to Hidden Data Role — "To hide a column, right-click and select Hide column or use the Roles pane to move it to the Hidden data role." Same thing!
❌ Security: Hidden is not for security — data is still in the query.

Q6 🔥: Which statement is TRUE about the Hidden Data Role?

✅ Cannot use with Data Tip Values — "You cannot assign a data item to the Hidden role and the Data tip values role at the same time." They're mutually exclusive.
❌ All objects: "Not available for all objects."
❌ Security: "Not intended for column-level security."
❌ Always increases: It should NOT increase rows.

Q7 🧠: "Role hides a ____. Page hides a ____. Column is a ____."

✅ data item / page tab / shortcut to Role — Hidden Data Role hides a data item, Hidden Page hides a page tab, Hide Column is just a shortcut to the Role.
