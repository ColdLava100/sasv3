❌ Mistake 1: Network Analysis — Ungrouped vs Hierarchy vs Targeted

Question: "Which type of network analysis object shows a series of linked nodes from a source to a target?"
You answered: d) Targeted ❌
Correct answer: c) Ungrouped ✅
Why C is right and D is wrong:
The course says: "An ungrouped network diagram creates a series of linked nodes from a source to a target."
"Targeted" is NOT a network analysis type at all. There are only TWO network structures:
• Ungrouped → Source → Target (linked nodes)
• Hierarchy → Parent → Children levels
🧠 Analogy — Family Tree:
• Ungrouped = "Who talks to who?" (any connection between any two people) — like a phone call network 📞
• Hierarchy = "Who reports to who?" (parent-child chain) — like a company org chart 🏢
• "Targeted" = Not a thing in VA ❌ (you made it up!)
✅ Memory Trick: "UNgrouped = UNstructured connections (any→any). Hierarchy = Has levels (parent→child)."

❌ Mistake 2: Network Analysis — Data Structure (Source ⊆ Target)

Question: "Which table is structured appropriately for ungrouped network analysis?"
You answered: d ❌
Correct answer: c ✅
The Rule You Missed:
"Target values must be a subset of Source values."

Look at the tables:
Option c ✅Option d ❌
Rows RACHEL→MONICA, RACHEL→ROSS, MONICA→(empty), ROSS→(empty) RACHEL→MONICA, RACHEL→ROSS, MONICA→CHANDLER, ROSS→JOEY

In option d: CHANDLER and JOEY appear as EMPLOYEE (target) but NOT as MANAGER (source).
❌ Sources = {RACHEL, MONICA, ROSS}. Targets = {MONICA, ROSS, CHANDLER, JOEY}.
❌ Targets contains CHANDLER and JOEY which are NOT in Sources → violates the rule!

In option c: All EMPLOYEE values (MONICA, ROSS) also appear as MANAGER values.
✅ Sources = {RACHEL, MONICA, ROSS}. Targets = {MONICA, ROSS}. All targets are sources. ✅
The empty cells just mean those managers have no direct reports — that's allowed.
🧠 Analogy — The "Everyone Works Here" Rule:
Think of it like a company: every employee must ALSO be someone's manager (or at least exist in the manager list).
You can't have an employee who doesn't appear in the employee directory at all!
Target ⊆ Source = "You can only be assigned as a subordinate if you also appear as a manager somewhere."
✅ Memory Trick: "Target must be Tiny part of Source. T ⊆ S. Every target must also be a source."
Check: List all unique sources. List all unique targets. Are all targets in the source list? If NO → wrong table.

❌ Mistake 3: Decision Tree — Data Item Types

Question: "What kind of data items can be used in a decision tree?"
You answered: a) Response = category only ❌
Correct answer: c) Response + Predictors = category, measure OR geography ✅
Why you got tricked:
You thought "Response must be a category" — but the course says:
"Response: Category, measure or geography."

The decision tree works with ALL 3 types for BOTH response AND predictors:
• Response = category, measure, OR geography ✅
• Predictors = any number of categories, measures, OR geographies ✅

The only restriction: ONE response + any number of predictors.
🧠 Analogy — The All-Accepting Tree:
The decision tree is like a friend who accepts everyone at their party:
• You can predict ANY type: category (Sports/Toys), measure ($50K), or geography (USA/Asia)
• You can use ANY number of predictors: 2, 10, 50 — doesn't matter
• The tree splits based on the data type automatically
✅ Memory Trick: "Decision tree = Doesn't discriminate — accepts ALL data types for BOTH response and predictors."
Role requirements: Response (mandatory) + Predictors (mandatory). Partition ID (optional).

❌ Mistake 4: Text Topics — Required Roles

Question: "What types of data items are needed to create a text topics object?"
You answered: a) ID + Document details + Language ❌
Correct answer: c) Unique row identifier + Document collection + Language ✅
You swapped "document collection" with "document details":
RoleTypeYour choice a ❌Correct c ✅
Document collectionCategory (contains text)❌ Missing!✅ Required
Unique identifierAny (row ID)✅ Had it✅ Required
LanguageCategory✅ Had it✅ Required
Document detailsOptional❌ Not required!Optional only

Document details is OPTIONAL (additional info like author, date).
Document collection is the REQUIRED one — it contains the actual text to analyze.
🧠 Analogy — Library Books:
• Document collection = The actual book with words/text 📖 (REQUIRED)
• Unique identifier = The book's ISBN number (REQUIRED)
• Language = English/Spanish/French (REQUIRED)
• Document details = Author, publisher, year (OPTIONAL — nice to have)
✅ Memory Trick: "Text Topics needs CIL — Collection (the text), Identifier (row ID), Language."
Document details is NOT required — it's extra!

❌ Mistake 5: Text Topics — Stem Words (+ sign)

Question: "For the second topic, what does '+drop' represent?"
You answered: a) Positive sentiment ❌
Correct answer: d) Stem words (drop, dropped, drops, dropping) ✅
Why the + does NOT mean positive:
The course says: "Different forms of a term (i.e. dropped and drop) are treated as the same term. They appear as +stem in topic bar chart."

The + before a word = stemming (showing the root word), NOT sentiment!
• +drop = includes drop, dropped, drops, dropping — all forms of the verb "drop"
• Sentiment is shown by the BAR COLORS (teal=negative, orange=neutral, green=positive)

Looking at the chart: Topic 2 (+drop, risk, supplier, +dissatisfy) has a HUGE teal (negative) segment — that shows the sentiment, not the + sign!
🧠 Analogy — Word Roots:
Think of +drop like a tree: "drop" is the ROOT, and different forms are the BRANCHES:
• 🌳 drop (root)
• ├── drop (present)
• ├── dropped (past)
• ├── drops (third person)
• └── dropping (continuous)

The + tells you: "Hey, we grouped all forms of this word together!"
The sentiment (positive/negative) is determined by the actual document content, not the +.
✅ Memory Trick: "+ before a word = +tems (stem). + does NOT mean positive."
Sentiment is in the BAR COLORS (teal=negative, green=positive), not the + sign ⚠️
