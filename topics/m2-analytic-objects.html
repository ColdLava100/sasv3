📈 3.1 Forecasting — Full Details

"A forecasting object uses the statistical trends in your data source to predict future data values."

AspectDetails
Data RequirementsTime series data item (date/datetime) + at least 1 measure
Default CI95% confidence — colored band around forecast line
Default Horizon6 periods
Binning IntervalAutomatic (default), Fixed count, or Use format
Models testedARIMA, Damped-trend exponential smoothing, Linear exponential smoothing, Seasonal exponential smoothing, Simple exponential smoothing, Winters (additive), Winters (multiplicative)
Model selectionVA automatically tests all models and selects the best one
DisplayHistorical values = markers only (no line). Hindcast (historical predicted) = part of forecast line. Forecast = solid line. CI = colored band.
Underlying FactorsAdditional measures that contribute to forecast. Each gets a weight (default 10%). Multiple factors possible.
What-If AnalysisScenario Analysis: Modify underlying factor values → see new forecast. Goal-Seeking: Set target forecast → find required factor values.
Measures LayoutAutomatic, Shared axis, or Separate axes
Optional RolesGroup, Data tip values

🔗 3.2 Network Analysis — Full Details

"Displays relationships by using a series of linked nodes."

AspectDetails
StructuresUngrouped: Source → Target linked nodes. Hierarchy: Parent-child levels using hierarchy role.
Required RolesSource (mandatory), Target (mandatory). Source must be a subset of Target values.
Optional RolesLinks (link width/color based on measure), Node size, Node color, Label, Data tip values
Metrics (Color)Betweenness Centrality: How often a node is on shortest path. Closeness Centrality: How close to all connected nodes. Community: Local groups. Disconnected Network ID: ID for each group of connected nodes.
Geo BackgroundIf Source+Target are geographic → can overlay on map (check Map background option)
OptionsNode size %, Link arrows, Link curvature, Abbreviate numeric values, Data labels. Successors count when source node selected.

🔀 3.3 Path Analysis — Full Details

"Displays flows of data — also called a Sankey diagram."

AspectDetails
Required RolesEvent (category — nodes), Sequence order (measure or date/datetime — orders events), Links
Link Color OptionsPath: Each path = separate group with distinct color (use for small # of paths). Drop-off: Colors by drop-off rate. Event: Colors by event. Sequence order: Colors by sequence.
Link WidthWeight or frequency (proportional to flow rate)
Path LengthMin & Max path length settings. Path ranking count and Compress options.
LayoutHorizontal or vertical
SummaryWhen maximized: Path Information tab (data behind diagram) + Path Analysis Summary tab (natural language explanation).
OptionsNode labels, Link labels, Nodes outline, Abbreviate numeric values (scale + digits), Display node titles

🌳 3.4 Decision Tree — Full Details

"An explainable and powerful ML algorithm using the C4.5 algorithm (iterative dichotomizer)."

AspectDetails
Required RolesResponse (category/measure/geography), Predictors (mandatory)
Optional RolesPartition ID, Frequency, Group, Label
AlgorithmC4.5 — uses entropy and information gain to select most relevant predictors
Interactive ModePrune branches, edit split rules, retrain. Zoom in to view leaf statistics. Right-click leaves for percent.
Leaf IDCan create new data item with leaf ID
General OptionsEvent Level: For category response — which level to model. Growth Strategy: Advanced option.
LicenseAdvanced decision tree options require additional license

📝 3.5 Text Topics — Full Details

"Performs sentiment analysis on text data — identifies topics and themes in unstructured text."

AspectDetails
Required RolesDocument collection (category — contains text), Unique identifier
Optional RolesLanguage, Document details
Max Topics2 to 25 topics
Cell WeightNone or Logarithmic
Term WeightSet weighting for terms
Maximized View3 tabs: Topics (bar chart + data), Topic Details, Sentiment Analysis
Use CasesCustomer feedback, survey responses, social media analysis

🎯 Key Exam Points — M2 L3:
• Forecasting: Default 95% CI, 6 periods. 7 models auto-tested. Binning: Auto/Fixed/Use format ⚠️
• Forecasting: Historical = markers only. Hindcast = part of forecast line ✅
• Network Analysis: Source must be subset of Target. 2 structures: Ungrouped & Hierarchy ⚠️
• Network metrics: Betweenness, Closeness, Community, Disconnected Network ID ✅
• Path Analysis: Event+Sequence order+Links required. Link color: Path/Drop-off/Event/Sequence order ⚠️
• Decision Tree: C4.5 algorithm. Response+Predictors mandatory. Interactive: prune, edit, retrain ✅
• Text Topics: 2-25 topics. Cell weight: None/Logarithmic ✅
• Auto Prediction: No manual model selection. Champion: accuracy (cat) or ASE (meas) ⚠️
