📤 Sharing & Embedding Reports — The Complete Guide ▶

From course: "You have many ways to share reports in Visual Analytics." This topic covers all methods for sharing reports — online (live connection) and offline (static exports).

📤 The Big Picture: Online vs Offline Sharing

🌐 ONLINE — Live connection to SAS Viya

✅ Interactive, live data
✅ Updates when data changes
❌ Viewer needs authentication (OR Guest Access)

Methods:
🔗 Share Report (within Viya)
🔗 Copy Link
🔗 Copy Embeddable Markup (SDK)
👤 Guest Access

📦 OFFLINE — Static exports, no connection

✅ No sign-in required
✅ Self-contained
❌ Static — data doesn't refresh
❌ No interactivity with SAS Viya

Methods:
📄 Export PDF
📦 Export Report Package
🖼️ Save Object Image

📊 Decision Tree: Which Sharing Method to Use?

🔍 All Sharing Methods — Detailed

🌐 ONLINE METHODS (Live connection to SAS Viya)

🔗 Share Report (within Viya)

Share with users within SAS Viya. Notification sent to their banner. Available in SAS Content folders.

🔗 Copy Link

Generate link to report, page, or objects. Can include Guest Access, parameters, banners. Interactive link.
"If recipient already signed in → report opens. If not → prompted for credentials or Guest button."

🔗 Copy Embeddable Markup (SDK)

Code snippet for SAS Viya SDK for JavaScript. Custom web page can embed reports.
"Custom web page must handle authentication, or each object displays sign-in screen."
Solution: Use Guest Access in the embed options.

👤 Guest Access

"Provides anonymous Read-Only access to select resources."
✅ No sign-in required for viewers
⚠️ Requires admin to configure in SAS Environment Manager
⚠️ Data must be in CAS with guest access granted
⚠️ ALL guests see same data — limit sensitive info!

📦 OFFLINE METHODS (Static exports, no Viya connection)

📄 Export PDF

Full report or selection as PDF.
⚠️ "If report contains private data, exporting to PDF is disabled."

📦 Export Report Package

"A snapshot of data and query results."
✅ Can view OFFLINE or host on a web server
✅ No sign-in required (static file)
❌ Not interactive — no live Viya connection
❌ "Private data disables report package export."

🖼️ Save Object Image

Export individual chart/object as image.
Quick way to capture a visualization.

👤 Guest Access — What You MUST Know

What is it?

"Guest Access provides anonymous Read-Only access to reports."
✅ Viewers don't need credentials
✅ Works with Copy Link and Embeddable Markup (SDK)
⚠️ Admin must enable it in SAS Environment Manager
⚠️ Grant Read access to SAS Content folders and reports
⚠️ ALL guests see same data — restrict sensitive info

Where is it used?

Copy Link: "Customize link with Guest Access"
Embeddable Markup: "Customize code with Guest Access"
Not used in: Exported Report Package (static, no Viya connection needed)

📦 Report Package — What You MUST Know

"You can export a report package, which is a snapshot of the data and query results for a report. You can then use the report package to view the report offline or host it on a webserver for others to access."

✅ Self-contained — no SAS Viya connection needed
✅ No sign-in — anyone can view
❌ Static — data doesn't refresh
❌ Private data — disables report package export

🎯 Exporting Objects — Filters & Images

Exam Question

Situation: A bar chart (MSRP by Type) has a filter action linked to a scatter plot (MPG City vs Highway). When "Wagon" is clicked on the bar chart, the scatter plot filters to show only Wagons. You export the scatter plot as an image.

Question: "Which statement is true?"

✅ D) Information about the applied filter action will not be displayed in the exported image.
❌ A) Filter removed, unfiltered shown — Wrong! Filter IS applied to exported image.
❌ B) Bar chart also exported — Wrong! Only the scatter plot is exported.
❌ C) Filter shown as appendix — Wrong! Images don't have appendices or metadata.

🧠 Key Rule

"Exporting an object as an image captures the visual state (filtered data), but NOT the filter/action metadata itself."

✅ Filtered data IS shown in the image
❌ Filter/action info is NOT included
❌ Other objects are NOT included
❌ No "appendix" or metadata is embedded

🔗 Embeddable Markup / SAS Viya SDK

"The copy embeddable markup feature generates a code snippet for a report, page, or object for use in the SAS Viya SDK for JavaScript."

Two authentication options:
1️⃣ credentials — Viewer must sign in (default)
2️⃣ guest — Viewer can view anonymously (if Guest Access is enabled)

"Custom web page must handle authentication, or each report object will display a sign-in screen."
