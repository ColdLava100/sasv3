🔗 1.1 Relationship Plots — Complete Guide

Located in the Graphs group. Include: Bubble plot, Correlation matrix, Heat map, Numeric series plot, Parallel coordinates plot, Scatter plot.

🫧 Bubble Plot

"Displays three dimensions of data for a group of category values."

AspectDetails
Required Roles3 Measures: X axis, Y axis, Size
Optional RolesGroup, Color, Lattice cols/rows, Data tip values, Animation, Hidden
⚠️ DefaultNon-aggregated — attempts one bubble per row! For large data, assign Group first
❌ CannotAssign Group AND Color simultaneously
AnimationData item in Animation role → player control (play/pause/scrub) shows changes over time
FocusFocus on bubbles that act differently (outliers)

🔢 Correlation Matrix

"Evaluates the linear relationship between multiple measures using Pearson's correlation coefficient."

AspectDetails
ThresholdsWeak ≤0.3 | Moderate 0.3-0.6 | Strong ≥0.6
Max Measures⚠️ 60
ModesWithin one set (all vs all, default) OR Between two sets (first = target)
⚠️ CriticalWeak correlation ≠ no relationship! Might be non-linear. Use scatter/heat map to check.
InteractionHover → data tip. Maximize → Details table. Right-click cell → New model from selection
OptionsShow border, Rotate axis labels

🔥 Heat Map

3 use cases: Two high-cardinality measures / Two categories / Category + measure
Axis Items: 2 items = single heat map. Multiple = matrix of heat maps (each pairing)
Options: Bin count, Transparency, Fit line (None, Best fit, Linear, Quadratic, Cubic, PSpline). Accepts Color role.
Purpose: Visualize trends, pinpoint outliers, identify nonlinear relationships ✅

📈 Numeric Series / 〰️ Parallel Coordinates / 🔵 Scatter

Numeric Series Plot: Shows measure at intervals after an event. 2 measures (X+Y). Optional: Group, Label, Lattice, Data tips, Hidden.
Parallel Coordinates: Multivariate data analysis. Each measure = vertical axis. Each observation = line. Color role highlights groups/clusters.
Scatter Plot: Two low-cardinality measures. Color: ≤8 values → unique colors, >8 → multiple marker shapes. Fit line: None/Best fit/Linear/Quadratic/Cubic/PSpline (Best fit uses backward variable selection).

🤖 1.2 Automated Explanation — Relationship Plot Selection Guide

"Determines the most important underlying factors that explain a particular variable."

3 Components: Natural language summary + Relative importance bar chart + Relationship plot
Roles: Response (mandatory), Time (optional), Underlying factors
Interaction: Click factor → relationship plot. Right-click → remove factor.

🔑 Relationship Plot Type — Depends on Variable Types

"Type of plot depends on variable type of both response and underlying factor."

📊 Measure underlying factor🏷️ Category underlying factor🌍 Geography underlying factor
📊 Measure ResponseScatter plot or Heat mapBar chartGeo region map or Geo coordinate map
🏷️ Category ResponseHistogram grouped by event level of interestStacked bar chart grouped by event level of interest(empty — no geo plot for category response)

🧠 Memory: Measure+M = Scatter/Heat. Measure+C = Bar. Measure+G = Geo. Category+M = Histogram. Category+C = Stacked bar.

🎯 1.3 Automated Prediction

"Predicts values of a response variable by specifying values for underlying factors."
Category response models: Logistic regression, Gradient boosting, Decision tree
Measure response models: Linear regression, Gradient boosting, Decision tree
Champion: Category = highest accuracy. Measure = lowest ASE.
⚠️ Cannot: be action source/target. Excluded from prompts. No manual model selection.
Roles: Response + Underlying factors. Auto-adds remaining data items (except aggregated measures, dates, calculated items, custom categories, derived items, frequency, hierarchies, filter/partition/spline/interaction effects).

📐 1.4 Containers — Full Comparison

"Containers are useful for grouping report objects. Located in the Containers group in the Objects pane."

ContainerPurposeLayout Options⚠️ Key Details
📦 Standard Arrange objects horizontally or vertically without overlap. Uses default grid layout. Container type: Flow (default), Stacking, Precision
Direction: Horizontal, Vertical (default)
Avoid scrollbars (selected): Resize content to fit (default) or Resize container to fit content		 Resizes content to fit. No scrollbars by default. Objects size to fill available space.
📚 Stacking Group related objects. Displays one object at a time. Takes same space. Easy to navigate. Container type: Flow, Stacking (default), Precision
Button type for nav bar		 Navigation bar to flip through objects. Labels = object names. Cannot format navigation bar. 3 uses: Group related objects, Alternative information (table + chart for same data), Pick your metric (different metrics).
📜 Scrolling "Functions like a web page." Shows scrollbar when content exceeds visible area. Container type: Flow (default), Stacking, Precision
Direction: Horizontal, Vertical (default)
Avoid scrollbars (deselected)		 Larger graphs, more info, better mobile experience. Test thoroughly on all devices.
🎯 Prompt Group control objects. Beneficial for mobile — easier navigation. Direction: Vertical or Horizontal. Width & height settings. Cannot define action to the prompt container itself. In prompt areas: define actions between controls. On canvas: define actions between controls+canvas objects. Delays actions until Apply. ❌ Prompt containers NOT included in PDF export.
🎨 Precision Create reports with overlapping objects. Control placement, alignment, depth, and size. Container type: Flow, Stacking, Precision (default) Often used to add image background. Most objects are transparent so background shows. ⚠️ Best practice: Set fixed report size. Can cause problems when reports are exported. ❌ Avoid precision containers for PDF export.

📦 1.5 Object Templates

How: Right-click object → Save as Template. Appears in Objects pane with default name. Saved without data → added to same group as original (e.g., bar chart template → Graphs group).
Where stored: SAS Content/Products/SAS Visual Analytics/Resources folder
Sharing: (1) Locate template in SAS Content, (2) Export via SAS Environment Manager (right-click → Export → JSON), (3) Import in target environment. Or export entire Resources folder for all templates + data views. CLI option via SAS Viya REST services.

🎯 Must-Know Exam Points — M2 L1:
• Bubble plot = NON-aggregated default. No Group+Color together ❌
• Correlation: Pearson's. Weak ≤0.3. Max 60. Weak ≠ no relationship ⚠️
• Relationship plot type depends on Response + Underlying factor variable types ✅
• Auto Predict: Cannot be action source/target. No manual model selection ⚠️
• Standard container: Flow(default)/Stacking/Precision. Avoid scrollbars by default ✅
• Stacking container: One object at a time. Navigation bar (cannot format). 3 uses ✅
• Scrolling container: "Functions like web page." Avoid scrollbars = deselected ⚠️
• Prompt container: NOT in PDF. Delays Apply. Cannot define action to itself ⚠️
• Precision container: Overlapping objects. Best practice: Set fixed report size ✅
• Object templates: Stored in Resources folder. Export individually or bulk ✅
