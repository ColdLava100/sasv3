📋 The 5-Step Process

Step 1 — Draft a Plan: Know your audience. What do they need? Select + prepare data. Create needed data items.
Step 2 — Choose the Best Chart: Match chart type to the message (see chart guide below).
Step 3 — Focus on What's Important: Use KPI objects. Use filters/ranks to reduce noise. Every element should serve a purpose.
Step 4 — Consider the Layout: Tell a single data story. Limit pages/objects. Use containers. Use neutral background. Avoid borders. Use default grid or containers. Consider placement of objects with actions. Use negative space.
Step 5 — Test, Test, and Test Again: "Test, Test, and Test Again." Sample audience. Verify interactions. Test on different devices. Review with stakeholders.

📊 Chart Selection Guide

Highlight a fact: Key value, Dynamic text, Donut chart, Faded bar chart
Compare (general): Bar chart, Dot plot (groups with close values), Butterfly chart (two groups across categories)
Compare (different ranges): Dual axis chart — ⚠️ might be misinterpreted! Fixed axis ranges useful for accessibility (sonification)
Compare (benchmark): Needle plot, Bullet gauge, Targeted bar chart, Display rules, Reference lines
Over time: Line chart, Time series, Step plot. Use line patterns for multiple lines (High Contrast theme).
Survey/results: Vertical bar chart, Crosstab, Box plots (provide explanation!)
Parts of whole: Pie (≤5 groups ⚠️), Donut, Treemap
Relationships: Scatter plot, Bubble plot, Heat map, Correlation matrix. Limit decimal places.
Processes: Path analysis, Network diagram
No graph needed: Word cloud, Image (specify alt text!), Text (avoid sensory characteristics), Data-Driven Content

♿ Accessibility Tips

• Use colors with different luminosity in gradients. Use default colors. Avoid 3-color gradients. Test with gray-scale mode.
• High Contrast theme: Override report theme. Include note about overriding. View line patterns and fill patterns.
• Avoid: animation, color-dependent display rules, lattices, linked selection actions, overview axes (screen readers can't interact).
• Provide meaningful titles and alternative text for accessibility.
• Screen readers cannot perceive highlighted elements (linked selection).
• Ensures legends display on all screen sizes.

👤 Viewer Capabilities

Comprehensive edits (default): Viewers can change object types, sorting, labels, legends.
Customize what viewers can do: export, print, comments, share. Configure in report options before publishing.
Auto control for prompts: Choose appropriate control based on type + cardinality. For hierarchies: control per level + cascading filters.

🎯 Key Exam Points — M2 L6:
• 5 steps: Draft → Chart → Focus → Layout → Test (Test Test Test) ⚠️
• Pie chart: Only for ≤5 groups. Hard to compare sizes ⚠️
• Dual axis: Can be misinterpreted. Fixed ranges for accessibility ✅
• Accessibility: Avoid animation, color rules, lattices, linked selection, overview axes. Use High Contrast theme ⚠️
• Just because you can doesn't mean you should — use simplest chart ✅
• Viewer capabilities: Comprehensive edits (default) → change types, sorting, labels, legends ✅
• Images: Must specify alternative text ⚠️
• Report Review pane available for testing ✅
