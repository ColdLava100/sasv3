❌ Your Mistake 1: Cell Naming — Letters = Columns, Numbers = Rows

Question: "What is the name of the highlighted cell in the graph template shown below?"
(A 2×2 grid with bottom-left cell highlighted in yellow)
You answered: b) B1 ❌
Correct answer: c) A2 ✅
Why you got confused:
You swapped columns and rows! The rule is:
• Letters = Columns (A, B, C...)
• Numbers = Rows (1, 2, 3...)

The bottom-left cell is:
• Column = left side = A ✅
• Row = bottom = 2 ✅
• Therefore = A2

B1 would be top-right (column B, row 1).
🧠 Memory Trick — The Alphabet Analogy:
Think of a spreadsheet:
A is the first column (left side). 1 is the first row (top).
So A1 = top-left, A2 = bottom-left, B1 = top-right, B2 = bottom-right.

"Letters = Left-to-right (columns). Numbers = North-to-south (rows)."
✅ Quick Check: Bottom-left = A2. Top-left = A1. Bottom-right = B2. Top-right = B1.
Just like Excel/Google Sheets naming!

❌ Your Mistake 2: Roles in Graph Builder

Question: "Which of the following is true about roles in SAS Graph Builder?"
You answered: a) You can assign data items to roles in graph builder ❌
Correct answer: b) Each object has certain required roles ✅
Why A is wrong and B is right:
Option A says "You can assign data items to roles" — this is true in a general sense but it's too obvious. The question asks about what's specifically true about Graph Builder.

Option B "Each object has certain required roles" — this is the more specific correct answer because in Graph Builder, when you create a template, you define which roles are required vs optional. Some roles are mandatory for the graph to work.

Also wrong:
• Option c) "You can only add one optional role" → ❌ You can add many optional roles
• Option d) "Each object has the same roles" → ❌ Different objects have DIFFERENT roles
🧠 Analogy — Building a Car:
• Option a = "You can put parts on a car" — obvious, not specifically about Graph Builder
• Option b = "Each car has certain required parts (engine, wheels, steering wheel)" — specific and correct! ✅
• Option c = "You can only add one extra feature" — wrong, you can add many
• Option d = "Every car has the same parts" — a truck ≠ a sports car
✅ Memory Trick: In Graph Builder, when you drop a graph element (like a bar chart), it has required roles (like Category + Measure) and optional roles (like Group, Color, etc.). You can specify which roles are required vs optional. That's the specific truth about Graph Builder roles!

❌ Your Mistake 3: Renaming Roles

Question: "Where can you rename roles in Custom Graph Builder?"
You answered: a) In the Options menu ❌
Correct answer: b) In the Roles menu ✅
Why B and not A:
The course says: "You can rename roles in the Roles menu, by clicking on the overflow menu (three dots) next to the role."

Think about it logically:
• If you want to rename a Role, where would you look? → The Roles menu (makes sense!)
• The Options menu is for visual properties (colors, fonts, sizes)
• The Outline menu is for the structure/layout hierarchy
• The Elements menu is for selecting graph elements
🧠 Analogy — The Right Tool for the Job:
• To rename a role → go to Roles menu (rename things where they belong) 🎯
• To change colors → go to Options menu 🎨
• To see the structure → go to Outline menu 🏗️

"Roles are renamed in the Roles menu — it's that simple!"
✅ Memory Trick: "To rename a Role, use the Roles menu. Three dots (⋮) → Rename."

❌ Your Mistake 4: Data-Driven Lattice

Question: "You create a custom graph template with multiple cells. If the number of cells can vary based on unique values of a class variable, what kind of template?"
You answered: a) A user-defined lattice ❌
Correct answer: d) A data-driven lattice ✅
The key difference:
TypeCells defined by...Number of cells
User-defined latticeYou manually set the rows & columnsFixed (e.g., 2×2, 3×1)
Data-driven latticeValues of a class variableVaries based on unique values ✅

The question says "number of cells can vary based on unique values" → That's the definition of a data-driven lattice!

User-defined lattice = YOU decide the layout (2 cols × 2 rows, fixed).
Data-driven lattice = The DATA decides (if your class variable has 5 unique values, you get 5 cells).
🧠 Analogy — Seating Arrangements:
• User-defined lattice = A wedding seating chart where you pre-define 10 tables of 8 seats each (fixed) 🪑
• Data-driven lattice = A restaurant where tables are added or removed based on how many guests show up (variable) 🔄

"User-defined = Uou choose (fixed). Data-driven = Data decides (variable)."
✅ Memory Trick: "If the number of cells varies with the data → It's data-driven. If you define the grid → It's user-defined."
