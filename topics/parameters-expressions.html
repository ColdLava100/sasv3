🎯 Parameters & Expressions — The Difference ▶

Parameters and Expressions are both created from the New data item menu, but they serve completely different purposes.

🎛️ Parameter

A VARIABLE whose value can be changed.

"A parameter is a variable whose value can be changed by report viewers."

Types: Numeric, Character, Date, Expression-based
User changes? ✅ Yes (via slider, dropdown, text input)
Think of it as: A knob/dial the user turns 🎛️

Examples:
🎛️ TopN = 10 (user slides to 25 → value = 25)
🎛️ SelectedRegion = "North" (user picks from dropdown)
🎛️ StartDate = 01JAN2025 (user picks a date)

📝 Expression

A FORMULA that calculates a result.

"The Expression Editor allows you to define values using operators, functions, and data items."

Types: Calculated Measure, Calculated Category, Calculated Date/Time
User changes? ❌ No (runs automatically on data)
Think of it as: An equation that computes 📝

Examples:
📝 Profit Margin = Revenue − Cost
📝 Full Name = First Name ‖ " " ‖ Last Name
📝 Age Group = IF Age < 18 THEN "Child" ELSE "Adult"

📊 How Parameters Flow Through a Report

🎛️ The 4 Parameter Types

🔢 Numeric Parameter

Viewer enters a number
"Top N count, rolling quarters, threshold values"
Controls: Slider, Text input, Drop-down, Button bar
Used in: Ranks, Display Rules, Filters, Calculated Items

🔤 Character Parameter

Viewer chooses a category/text
"Select a region, choose a product category"
Controls: Text input, List, Drop-down, Button bar
Used in: Calculated Items, Filters, URLs

📅 Date Parameter

Viewer selects a date
"Choose start date, view 30 days from selected date"
Controls: Slider, Text input, Drop-down, Button bar
Used in: Filters, Calculated Items, Time series plots

🧮 Expression-Based Parameter

Value set by a formula, not user input
"Average profit across ALL data (unaffected by filters)"
No control needed — formula determines value
Used in: Benchmarking, % of Total, Initial values

Pass parameter value in external URL

🎯 Real Scenario: Combining Parameters + Expressions

Step 1: Create Parameter

Numeric Parameter called TargetProfit
Initial value: $50,000
Add a Slider control → user can slide from $0 to $100,000

Step 2: Create Expression

Calculated Item: Profit vs Target
Formula: Profit - TargetProfit
This uses the parameter TargetProfit in the expression!

Step 3: See It In Action

Bar chart shows "Profit vs Target" per region

User slides target to $80,000 → all bars recalculate
User slides target to $30,000 → bars change again

Parameter controls the input. Expression calculates the result.

🧠 Expression-Based Parameters — The Tricky One:
"Expression-based parameters can be used anywhere a regular parameter can be used. They are NOT affected by filters or actions — based on the entire data source, value remains constant."

Use for: Benchmarking (average across all data), Percent of total, Setting initial slider values to "past 180 days" automatically.
Example: Create AvgOrderProfit = SUM(Profit) / SUM(Orders). Even if you filter to "North" region, this parameter still shows the average across ALL regions — perfect for comparison!
🧠 Mnemonic:
Parameter = Put in a value (user controls it) 🎛️
Expression = Evaluate a formula (SAS calculates it) 📝
Expression-based Parameter = Formula becomes the Put-in value 🤝

"Parameters are the INPUT. Expressions are the CALCULATION. They work TOGETHER."
