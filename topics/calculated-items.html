🧮 Calculated Data Items & Aggregated Measures

🧮 Calculated Data Item vs Aggregated Measure — The Core Difference

Both are created from the New data item menu. But WHEN they calculate is different.

🧮 Calculated Data Item

Evaluated on: Detail data (row by row)
Aggregation function? ❌ Not required
Preview results? ✅ Yes
Result type: Measure, Category, or Date/Time
Scoped expression? ❌ No

"For a calculated data item, the expression is evaluated on detail data."

Example: Revenue − Cost
→ Each row: $100 − $60 = $40. Then SAS aggregates the results.
→ Good for: Adding surcharges, creating profit margins, concatenating names

📊 Aggregated Measure

Evaluated on: Aggregated data (after grouping)
Aggregation function? ✅ REQUIRED (SUM, AVG, COUNT, etc.)
Preview results? ❌ No
Result type: Always Aggregated Measure
Scoped expression? ✅ Yes (different formulas per category)

"For an aggregated measure, the expression is evaluated on aggregated data."

Example: Profit % of Total
→ First group and SUM: Sports=$300, Toys=$200. THEN divide: 300÷500 = 60%
→ Good for: Percent of total, period differences, running totals

"Aggregate first, then calculate."

🧠 Quick Decision Guide

Use Calculated Item when:
→ Adding a surcharge to each sale
→ Creating "Full Name = First + Last"
→ Revenue − Cost per product

Use Aggregated Measure when:
→ Each product's % of total sales
→ Difference from previous period
→ Running totals or cumulative sums

"Calculate first, then aggregate = Calculated Item. Aggregate first, then calculate = Aggregated Measure."

⚙️ Aggregated Measure Contexts

Two contexts when defining a formula:

By Group — Calculates per subset of data
SUM(ByGroup, Profit) → sums profit for each product

For All — Calculates for entire data (after filtering)
SUM(ForAll, Profit) → total profit across everything

Combined: SUM(ByGroup, Profit) / SUM(ForAll, Profit) = % of total

📅 Periodic Aggregated Measures — Time-Based Calculations

Periodic functions aggregate values over time. From course: "Periodic functions are evaluated over time intervals, allowing you to analyze how data changes over time. To use the periodic functions, your data source must contain date or datetime data items."

5 Periodic Functions

CumulativePeriod — Current + all previous periods within a larger period
ParallelPeriod — Value from a parallel time period (e.g., same month last year)
Period — Value for a specific period (e.g., current month)
PeriodWithDate — Value for a specific, constant period
RelativePeriod — Value relative to current period (e.g., 3 months ago)

Filter Options

ApplyAllFilters — All filters applied before processing
IgnoreAllTimeFrameFilters — Date/time filters ignored
IgnoreInteractiveTimeFrameFilters — Only interactive date filters ignored

🎯 Exam Question: Percent Difference in a Crosstab

Situation: An aggregated measure "Product Sale (Percent diff from previous period)" was created using the Year data item. It's added to a crosstab with City in Rows. The column shows dashes "—".

Question: "Which data item should be added to the Columns role to correctly display the percent difference?"

✅ C) Year — The period dimension must be in the table structure

🔍 Why Year is Correct

The aggregated measure was created "using the Year data item" — Year defines the periods. For a periodic function like Percent Difference to calculate values, the period dimension must be present in either Columns or Rows of the crosstab.

Without Year in Columns:
┌──────┬──────────┬───────────────┐
│ City │ Sales │ % Diff (—) ❌ │
├──────┼──────────┼───────────────┤
│ Fresn│ $193,659 │ — (no period) │
└──────┴──────────┴───────────────┘

With Year in Columns:
┌──────┬───────┬───────┬───────┐
│ City │ 2022 │ 2023 │ %Diff │
├──────┼───────┼───────┼───────┤
│ Fresn│$180K │$193K │ +7.2%✅│
└──────┴───────┴───────┴───────┘

❌ Why NOT the Others

A) Quarter — Period is YEAR, not quarter. Wrong granularity.
B) Month — 226 distinct months, too granular. Also wrong period type.
D) Date — 4,900 distinct dates! Would create thousands of columns.

The period dimension must match what the aggregated measure was created with.

🧠 Key Rule: "Periodic functions need their period dimension in the table structure. When the % Diff shows dashes, the period is missing from Columns or Rows."
⚠️ Exam Tip: In the expression editor, aggregated functions do NOT accept aggregated measures as inputs (no nested aggregations). Also, "Min and Max aggregated measures return a date/datetime formatted aggregated measure — it cannot be used directly in filters."
