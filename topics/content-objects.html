🌐 Web Content

"Useful for adding references and video instructions to your report."

✅ Display web page (URL)
✅ Embed video in an iFrame
❌ Websites that block iFrame won't show
❌ Cannot be source of filter action
❌ Not included in PDF export

⚙️ Job Content

"Enables you to access the functionality of SAS code and display the output in your report."

✅ Access any data
✅ Create output
✅ Update tables
✅ Works with or without forms
✅ Legacy job support
❌ Cannot be source of filter action

📊 Data-Driven Content

"Enables you to utilize custom third-party visualizations in your report."

✅ Uses D3.js, Google Charts, CanvasJS, etc.
✅ Receives data from VA (JSON format)
✅ Can be source of action
✅ Responds to filters/ranks/actions
❌ Requires HTML, CSS, JS knowledge
❌ Not included in PDF export

📨 Data-Driven Content — 3 Message Types

📝 Instructional

Data-Driven Content → VA
"Type, number, and order of required data items."

📊 Data

VA → Data-Driven Content
"Data assigned to object and metadata in JSON format."

🔘 Selection

Data-Driven Content → VA
"Details about selections, used when source of action."

🎯 Key Exam Points:
• Only Data-Driven Content can be a source of action ✅
• Only Data-Driven Content receives data from VA ✅
• Web Content and Job Content CANNOT be source of filter action ❌
• Web Content and Data-Driven Content are NOT in PDF export ❌
• Web Content uses iFrame — some websites block this ⚠️
• Data-Driven Content requires HTML, CSS, and JavaScript ⚠️
🧠 Memory Trick:
Web = Website URL (just paste!)
Job = Just SAS code (pre-built)
Data-Driven = Dynamic data + code (most powerful!)

"Web is a link, Job runs code, Data-Driven does both AND talks back to VA."
