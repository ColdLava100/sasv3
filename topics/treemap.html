🟦 Treemap — Roles Explained CHARTS ▶

A treemap shows part-to-whole relationships using nested rectangles (tiles). Each tile = a category, sized by a measure, colored by another measure.

📋 How Treemap Roles Work

Visual ElementWhat it showsRole in SAS
🏷️ Tiles (rectangles)Which categories existCategory (Product Line)
📏 Tile SizeHow big each tile isSize measure (Revenue) or Frequency
🎨 Tile ColorColor gradient across tilesColor measure (Profit)

🎯 Exam Question

Question: "Profit is currently assigned to Color. If Profit were removed from this role, what would happen?"

❌ Your answer: Tiles colored by frequency of Product Line
✅ Correct: The tiles would all be the same color.
Why? No Color role assigned = no data driving colors = all tiles default color. Color and Size are independent roles — removing Color doesn't affect Size.

⚙️ Key Takeaways

Category
Defines the tiles/groups.
Always needed.
Size
Controls tile area.
Defaults to Frequency if empty.
Color 🎨
Controls tile color gradient.
If empty → all same color.
🧠 Memory Trick:
Category = which tiles exist
Size = how big each tile is
Color = what color each tile is

"If you remove Color, you lose the rainbow — not the sizes. Size and Color are independent."
