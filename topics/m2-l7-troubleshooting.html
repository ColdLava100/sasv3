❌ Your Mistake: PDF Export — What's Included vs Not

Question: "Which of the following is true about exporting a report as a PDF?"
You answered: a) You cannot expand clipped content for list tables and crosstabs ❌
Correct answer: c) Filters are displayed in an appendix ✅
Why a is wrong: You CAN expand clipped content! The course actually mentions "expand clipped and non-visible content" as an important feature in PDF export that prints all returned rows within the data threshold.

Why c is correct: "By default, an appendix is included in the PDF if the report, page, or object has descriptions, filters, warnings, errors, or display rule legends."
Filters, descriptions, warnings, errors, display rule legends = ALL included in the PDF appendix.

📋 Complete PDF Export Reference — What's IN vs OUT

✅ INCLUDED in PDF

  • Bar charts, tables, graphs ✅
  • Filters (in appendix) ✅
  • Descriptions (in appendix) ✅
  • Warnings & errors (in appendix) ✅
  • Display rule legends (in appendix) ✅
  • Selected object in Container objects ✅

❌ NOT INCLUDED in PDF

  • Data-Driven Content ❌
  • Web Content ❌
  • Hidden pages ❌
  • Report prompts ❌
  • Page prompts ❌
  • Prompt containers ❌
⚠️ Other PDF considerations:
• Only selected object in Container objects is printed
• Scrollable content may not be printed
• Fonts may not match (if fonts used don't match server fonts)
• The print feature attempts to fill available space on the PDF page
• You CAN expand clipped content for list tables and crosstabs ✅
• Items NOT in PDF: Data-Driven Content, Web Content, Hidden pages, Report prompts, Page prompts, Prompt containers
🧠 Memory Trick — "The 6 Excluded from PDF":
Data-Driven Content, Web Content, Hidden pages, Report prompts, Page prompts, Prompt containers
"DW HR PP — Data Web, Hidden Report, Page Prompt — all left out of PDF!"

Everything else (filters, descriptions, warnings, legends) = in the appendix ✅

📤 Online vs Offline Sharing — Complete Reference

MethodOnline/OfflineKey Details
Share Report🌐 OnlineWithin SAS Viya. Recipients must have SAS account. Notification sent to banner. Available in SAS Content folders.
Guest Access🌐 OnlineAnonymous read-only access. Admin must enable via SAS Environment Manager. Guest is an internal user to Viya.
Copy Link🌐 OnlineInteractive link at report/page/object level. Customize with guest access, banner, parameters. If not signed in → prompt for credentials or Guest button (if enabled).
Copy Embeddable Markup🌐 OnlineCode snippet for SAS Viya SDK for JavaScript. Embed in custom web pages. Customize with guest access, interactive/static, banner.
Export PDF📥 OfflineFull report or selection. Appendix includes filters/descriptions/warnings/errors/legends. ❌ Not included: DW HR PP.
Export Report Package📥 OfflineSnapshot of data + query results. View offline or host on web server. No Viya needed.
Save Object Image📥 OfflineIndividual object as PNG. Default size = report size. ❌ Does NOT display applied ranks or filters. Special chars → hyphen.
Export Data📥 OfflineCSV, TSV, or Excel. No totals for crosstab/list table. Analytics objects → separate worksheets per element.
Distribute Report📤 AlertEmail to recipients. Optional: Microsoft Teams, attached PDF, summary. Recurring schedule (daily, weekly, etc.)
Generate Alerts📤 AlertReport objects must have a Display Rule defined. Alert triggered when condition met. Recipients are registered Viya users.

🔒 Private Data — Export Restrictions

If private data is present:
• Report → PDF export and Report Package export are disabled ❌
• Page → PDF export is disabled if page contains private data ❌
• Object → Data, image, or PDF export are disabled ❌
• Detail data export → always disabled if private data in data source ❌
• Additional columns not used in object may still enable inference → disabled ⚠️

Viewer Capabilities:
• Comprehensive edits (default) — viewers can change object types, sorting, labels, legends
• Viewer Capabilities option in Options pane at report level (introduced in VA 8)
• Cannot be assigned through CLI ❌
• Export controls: Enable / Disable if private data / Always disable ✅

👤 Viewer Capabilities Levels

Comprehensive edits (default): Viewers can change object types, sorting, adding labels, hiding/displaying legends ✅
Controls available: export, print, comments, share — configured in report options before publishing.

For accessibility: Use High Contrast theme. Avoid animation, color-dependent rules, lattices, linked selection, overview axes.

🌍 Localize & Identify Issues

Localize Reports: Add translations for multiple languages. Use ISO 639 two-letter language code + country code. Can use interactive UI or CLI to export translations, edit, then import.

Identify Issues: Use Issues pane (Report Review pane) to find: data loading problems, missing items, broken links. Can export report review as PDF to track improvements.

Data Thresholds: Handle large data. Set limits. Warn when exceeded. Expand clipped content in PDF (all returned rows within threshold).
