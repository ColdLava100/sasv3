🎮 Topic 9: Designing Interactive Reports — Prompts, Actions & Links ▶

From course: "Prompts and actions can make your reports more versatile by enabling viewers to access the information that's important to them."

🚗 The Car Dashboard Analogy

Prompts 🎛️ = Climate control — viewer sets temperature to their liking
Actions 🔗 = Steering wheel — click one thing, another responds
Links 🔗 = GPS — tap to navigate somewhere else

🔗 Actions — The Core Concept

Actions let one object talk to another when you click. Think of it like a steering wheel 🚗 — turn it and the wheels respond.

Two types: Automatic (between ALL objects) vs Manual (between specific objects). Cannot combine both on same page ⚠️

Feature⚡ Automatic✋ Manual
ScopeBetween ALL objects on pageBetween specific objects you choose
SetupOne setting = everything connectedAdd one-by-one
Combine?❌ Cannot combine auto + manual on same page
AnalogyOpen-plan office 🏢Walkie-talkie 🎙️

⚡ 3 Automatic Action Types

🔗 Linked Selection

"Highlights related data in multiple objects."

🖍️ Click → related data highlights (blue)
✅ Other rows stay visible (grayed)
🔄 Applied in any order
🏢 Analogy: Highlighter pen

➡️ One-Way Filter

"Enables viewers to apply filters between objects in any order."

🔽 Click → other objects filter
✅ Hierarchical / cascading
🏢 Analogy: Waterfall 🌊 — flows one way
📋 Default in: Dashboard, Panel, Stack templates

⬅️➡️ Two-Way Filter

"Applies filters between ALL objects, even previously selected ones."

🔄 Click → filters ALL objects
⚠️ Do NOT use with required prompts
🏢 Analogy: Echo 🔊 — affects everything
❌ Can get stuck if + required prompts

✋ Manual Actions — 2 Types

TypeWhat it doesSource → Target?Best for
FilterSubsets data in target object🔽 UnidirectionalClick a map → table shows only that region
Linked SelectionHighlights data in target object🔽 UnidirectionalClick a bar → related rows light up in the table

⚠️ Filter Action Rules

✅ CAN be source/target

Bar chart, table, geo map, KPI, controls (dropdown, slider*), etc.
*Slider with "Set fixed range" cannot be source

❌ CANNOT be source

Images, Job Content, Text, Web Content

🔑 Key Properties:
1. Unidirectional 🔽 — Source → Target only (not backward)
2. Transitive 🔄 — A filters B, B filters C → A also filters C
3. Different data sources — Need to map data sources first 🗺️

🔗 Links — Navigation

Link TypeGoes to...AnalogyNotes
📄 Page LinkAnother page in same reportPage turn 📖Includes hidden pages (viewer sees as normal)
📊 Report LinkA different reportReference book 📚Opens separately
🌐 URL LinkAn external websiteBrowser tab 🌍URL must be accessible
🔙 Return LinkBack to previous pageBack button ↩️Auto-added when linking to basic/hidden pages
🎯 Key Exam Points:
• Auto + Manual actions CANNOT be combined on same page ❌
• Linked Selection = highlights. Filter = subsets data ⚠️
• All filter actions are unidirectional and transitive 🔄
• Images, Text, Web Content, Job Content = cannot be source ❌
• All controls (dropdown, slider, etc.) can be source AND target ✅
• Slider with fixed range = cannot be source ⚠️
• Required prompts + 2-way filter = can get stuck ❌
• Hidden pages appear normal when accessed via link ✅
• New pages default to manual actions (not auto) ⚠️
• Best practice: Finalize layout before enabling automatic actions ✅
🧠 Memory Trick:
Prompts = Pick what to see (viewer chooses)
Actions = Objects Answer each other
Links = Leave this page (navigate)

"Prompts are the TV remote 📱, Actions are the steering wheel 🚗, Links are the GPS 🗺️."
