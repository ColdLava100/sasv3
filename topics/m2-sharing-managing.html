📤 M2 L7: Sharing & Managing Reports▶

📤 Online Sharing

Share Report: Within SAS Viya — recipients must have SAS account
Guest Access: Requires admin to enable in SAS Environment Manager
Copy Link: Can include guest access token, parameters, banners
Copy Embeddable Markup: SDK for JavaScript — embed SAS report in custom web page
Distribute/Send Alerts: Email report links or data — recipients must be registered Viya users

📦 Offline Sharing

Export PDF: Full report or specific pages/objects. ❌
Export Report Package: Self-contained snapshot of data + queries — no Viya needed. Filters displayed in appendix.
Save Object Image: Individual object as image file. Captures visual state but NOT metadata about filters/actions.
Export Data: Underlying data as CSV or Excel. Only selected object in container objects is printed.

👤 Viewer Capabilities & Management

Viewer Capabilities: Customize what viewers can do — export, print, comment, share. Set in report options before publishing.
Data Thresholds: Handle large data gracefully. Set limits. Warn when exceeded.
Localize Reports: Translate for different locales — date formats, currency, language.
Identify Issues: Use Issues pane in edit mode to find: data loading problems, missing items, broken links.

🎯 Key Exam Points — M2 L7:
• ❌ NOT in PDF: Data-Driven Content, Web Content, Hidden pages, Report prompts, Page prompts, Prompt containers ⚠️
• Object Image: Shows filtered data but NO filter metadata ✅
• Report Package: Self-contained snapshot — no Viya needed ✅
• Guest Access: Must be enabled by admin ⚠️
• Alerts: Recipients must be registered Viya users ✅
• Export Data: Only selected object in containers is printed ⚠️
